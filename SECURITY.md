# 安全政策

## 支持的版本

| 版本 | 支持状态 |
| ---- | -------- |
| 1.0.x | ✅ 支持 |
| < 1.0 | ❌ 不支持 |

## 报告漏洞

如果您发现安全漏洞，请**不要**通过公开的 Issue 报告。

相反，请通过以下方式联系我们：

1. **GitHub Security Advisories**（推荐）
   - 访问项目的 Security 页面
   - 点击 "Report a vulnerability"
   - 填写详细信息

2. **邮件联系**
   - 发送邮件至项目维护者
   - 标题请注明 "[Security] 单词小达人"

### 报告内容

请包含以下信息：

- 漏洞类型
- 复现步骤
- 影响范围
- 可能的修复建议（如有）

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：7 天内
- **修复发布**：根据严重程度，30 天内

## 安全最佳实践

### 对于用户

- 仅从官方渠道下载应用
- 保持应用更新到最新版本
- 不要安装来源不明的修改版

### 对于开发者

- 定期更新依赖库
- 遵循安全编码规范
- 不要在代码中硬编码敏感信息
- 使用环境变量管理 API 密钥
